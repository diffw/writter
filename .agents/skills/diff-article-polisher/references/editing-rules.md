# Editing Rules

## Absolute Rules

- Preserve original meaning, viewpoint, values, and faith stance.
- Preserve the author's personal language habits as much as possible.
- Edit only the article body.
- Never modify the Markdown frontmatter/header block.
- Prefer minimal necessary intervention.

## Allowed Changes

- Correct typos and obvious transcription mistakes.
- Smooth sentence order when the current order blocks comprehension.
- Re-segment paragraphs for better breathing space.
- Remove accidental duplication caused by spoken drafting.
- Add or adjust punctuation when it clarifies rhythm and meaning.
- Keep Markdown structure intact while improving readability inside paragraphs or list items.

## Sentence-Level Standards

- Remove semantic overlap before attempting stylistic polish.
- If two nearby images have an inclusion relationship, usually keep the sharper one instead of both.
- In one sentence, keep the core action chain as stable as possible.
- If a sentence begins with observation, do not repeatedly switch between `看`, `看到`, `想到`, `感到` unless the shift is meaningful.
- Prefer concrete visible images over idiomatic abstractions when the sentence is describing a scene.
- Let a descriptive sentence unfold along one visual, spatial, or emotional path.
- Preserve repetition only when it adds rhythm, escalation, focus, or sincerity.
- Remove repetition when it merely restates the same semantic unit with no added force.

## Repetition Test

When evaluating repetition, ask:

1. Does the repetition create rhythm?
2. Does it narrow or widen the image in a clear direction?
3. Does it intensify emotion or focus?
4. Or does it simply restate overlapping content?

If the answer is mostly the fourth case, compress it.

## Disallowed Changes

- Changing claims, conclusions, convictions, or emotional direction
- Reframing the piece into a more neutral, safer, softer, or more “correct” opinion
- Replacing the author's candid voice with generic literary prose
- Converting all spoken phrasing into textbook written Chinese
- Standardizing away rhetorical questions, pauses, parentheses, and emphasis
- Rewriting the header, tags, dates, title, quote formatting, image links, or metadata

## Voice-Transcript Handling

When the draft is obviously produced from speech:

- Keep the speaking rhythm unless it becomes hard to read.
- Remove filler only when it is noise rather than style.
- Turn one long spoken block into several paragraphs if that makes the piece readable.
- Retain self-questioning, repeated emphasis, and conversational pivots if they carry the author's presence.
- Avoid “upgrading” the piece into a formal article that the author would not naturally write.
- Do not preserve transcript noise just because it was spoken; distinguish live rhythm from accidental clutter.

## Style Signals To Preserve

- First-person introspection
- Frank self-exposure
- Movement from concrete scene to inner reflection
- Frequent use of `我想` / `我觉得` / `其实` / `但是` / `比如` / `于是`
- Rhetorical questions and self-questioning
- Parentheses and em-phatic aside-like inserts
- Selective English terms, names, product names, or timestamps when naturally present
- Numbered lists for gratitude, takeaways, or structured reflection

## Self-Check Before Finishing

Check each item:

1. Did I change any viewpoint, value judgment, or faith expression?
2. Did I accidentally over-formalize the text?
3. Did I erase authentic roughness that should remain?
4. Did I keep the frontmatter exactly unchanged?
5. Did I preserve the article's original structure unless a local change clearly helped?
6. Would the author still sound like himself if he read this aloud?
